# Overlay Registry — Long-Horizon Strategy

*Append-only institutional memory for every overlay, sell rule, replace rule, and
universe/data change ever tested against the long-horizon (3-month cross-sectional
trend-momentum) strategy. This is the living ledger that extends
[`STRATEGY_FULL.md` §11](../long_horizon/STRATEGY_FULL.md) — §11 is the prose narrative;
this table is the structured, queryable record future sessions consult **before**
proposing anything.*

---

## Why this file exists

The strategy's edge is **rule-based clarity** on a clean universe with disciplined exits.
Every optional overlay tested so far has either failed to improve risk-adjusted return or
actively hurt it. That is not a failure of imagination — it is the signal that the core is
already near its ceiling, and that new overlays carry **skeptical priors by default**.

The governing discipline:

> **A reversed or null result is a FINDING, recorded so it is not re-tried without
> genuinely new information.** "New information" means: **new data** (a source we did not
> have), **a new feature** (orthogonal signal we have not computed), **a new sub-period**
> (the forward wall has advanced and we now have out-of-sample evidence the old test could
> not see), or **a genuinely different formulation** (not the same test re-run the same
> way). Re-running a dead overlay because it "feels like it should work" is forbidden — the
> point of this ledger is to make that cost zero by remembering.

## How to use it

1. **Before proposing** any sell/replace/sizing/risk/universe overlay, search this table.
   If the id or a near-identical hypothesis is already `REJECT`, you may not relitigate it
   without stating which of {new data, new feature, new sub-period, new formulation} you
   bring. Cite it in the `mechanism note`.
2. **After the harness tests it**, append a row. Never edit or delete a prior row — if a
   verdict is overturned by new evidence, add a **new** row that references the old id (e.g.
   "supersedes O-003 — forward wall advanced to 2027"). Append-only is the whole point.
3. **Verdicts** use the promotion bar from
   [`skills/sell-replace-logic/SKILL.md`](../skills/sell-replace-logic/SKILL.md):
   - **PROMOTE** — ALL hold: post-tax post-cost ΔSharpe ≥ +0.10, ΔCalmar ≥ +0.05,
     positive ΔCAGR in the 2022–2026 sub-period, walk-forward fold-pass ≥ 60%, bootstrap
     95% CI on ΔSharpe excludes 0, turnover increase ≤ 30%, mechanism explainable in one
     sentence.
   - **SHADOW** — 4–5 of the bar hold (log the signal, touch no trade).
   - **REJECT** — otherwise.
   - **OPEN** — proposed/sequenced but not yet measured through the harness (no verdict).
4. **The baseline everything is measured against** is the FROZEN-cfg arm (corrected-682 universe, 397 solvent names):
   **Gross: 26.1% CAGR / 1.02 Sharpe / −41.9% maxDD** (research/baseline_v0.json, 2026-06-27)
   **After-tax (STCG 20%): 23.1% CAGR / 0.83 Sharpe** — the honest, reproducible anchor.
   *(Previously reported as 30.26% / 1.15 — that was optimistic-exit measurement, superseded by baseline_v0. The ~4pp CAGR gap reflects exit-parity unification: backtest now fills targets conservatively, matching live. Walk-forward "≥2019 ~32% / 1.31" was also optimistic-exit, now pending Stage-B re-confirmation.)*
   Do **not** anchor on the re-derived 34.67% / 1.248 variant (`_tradelog_682.json`); that
   is a different parameterisation (see W-21 in the wiring ledger).

## ID scheme

- `O-###` — overlays / risk / sizing layers (Phase 6 sizing/exit/risk; the §11 rejections).
- `S#` / `R#` — the sell-replace candidates from `skills/sell-replace-logic/SKILL.md`
  (S1–S7 sell-beyond-mechanical, R1–R4 rotation). Ids match the skill so the two documents
  cross-reference cleanly. These are **OPEN** until the Phase 4 harness tests them.
- `U-###` — universe / data overlays (the deferred Phase-1 audit items).

All "result" cells state the **actual measured effect if known**; for OPEN rows the result
column reads `— (not yet measured)`. Numbers are cited to their source file. **Never invent
a result** — an untested row stays OPEN.
